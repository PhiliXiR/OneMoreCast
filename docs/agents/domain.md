# Domain Docs

This is a single-context repository.

## Before exploring

- Read the root `CONTEXT.md` for canonical domain language.
- Read relevant decisions under `docs/adr/`.
- If either is absent, proceed without requiring it to be created upfront.

## Vocabulary

Use terms defined in `CONTEXT.md` in issue titles, specifications, tests, and
implementation discussions. Avoid synonyms that the glossary explicitly marks
as discouraged.

## Architectural decisions

Surface conflicts with an existing ADR explicitly rather than silently
overriding the recorded decision.
